<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Draft markdown -> Anthropic request | Writer-authored document content is embedded into the AI Review request and must be treated as analyzable content, not instructions. | Full document text and user review prompt |
| Main process -> worker runtime | Review jobs are serialized and executed in the worker loop, which must not regress into malformed or runaway tool behavior. | Request payload, tool calls, tool results |
| Controller prompt policy -> review output | Persona and review-policy instructions shape what comments are emitted and where they land. | Persona choice, tool instructions, comment-scope guidance |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Denial of Service | AI review request sizing | mitigate | Set explicit per-job token ceilings so AI Review has enough headroom for multi-comment tool responses while keeping existing loop guards intact. |
| T-01-02 | Tampering | AI review prompt policy | mitigate | Remove the outdated fixed comment-count instruction and keep explicit tool-only/no-rewrite constraints in the prompt. |
| T-01-03 | Tampering | Persona routing and session history | mitigate | Add runtime assertions that persona-specific system instructions and conversation history persistence continue to work after the request-construction changes. |
</threat_model>
