<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Toolbar/popover -> Vuex runtime state | User intent to run AI review must become a single active review session, not duplicate launches. | Persona choice, optional prompt, button state |
| Main-process refresh -> renderer overlays | Persisted margin comments are turned into visible rail cards and dots. | Thread anchors, resolved paragraph positions |
| Scroll position -> overlay layout | Card and dot placement depend on live DOM geometry and must stay aligned while the editor moves. | `getBoundingClientRect()`, `scrollTop`, paragraph IDs |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | Have a Look trigger path | mitigate | Disable visible review controls during running state so the writer cannot accidentally launch the same review twice. |
| T-02-02 | Tampering | Margin overlay positioning | mitigate | Refresh rail cards and dots on editor scroll through the existing RAF-throttled layout path instead of leaving stale positions on screen. |
| T-02-03 | Repudiation | Mixed-scope review surfacing | mitigate | Add a real toolbar-path e2e test that proves global and margin review results appear in their intended UI surfaces. |
</threat_model>
